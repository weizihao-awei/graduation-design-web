\# 私信功能前端开发API文档



\## 基础信息



\- \*\*Base URL\*\*: `/api/message`

\- \*\*Content-Type\*\*: `application/json`

\- \*\*认证方式\*\*: 需要登录，使用Token认证



有一个要求就是，websocket连接只在前端在si'x界面时开启，用户离开私信界面应当立即断开。

\\\*



\## 统一响应格式



所有接口返回统一的响应格式：



```json

{

&#x20; "code": 200,

&#x20; "message": "操作成功",

&#x20; "data": {},

&#x20; "timestamp": 1234567890

}

```



\- `code`: 状态码，200表示成功，其他表示失败

\- `message`: 响应消息

\- `data`: 响应数据，具体结构见各接口说明

\- `timestamp`: 时间戳



\*\*\*



\## 接口列表



\### 1. 获取或创建会话



\*\*接口地址\*\*: `POST /api/message/chat/get-or-create`



\*\*功能说明\*\*: 根据对方用户ID获取会话ID，如果会话不存在则自动创建



\*\*请求参数\*\*:



```json

{

&#x20; "otherUserId": 123

}

```



| 参数名         | 类型   | 必填 | 说明     |

| ----------- | ---- | -- | ------ |

| otherUserId | Long | 是  | 对方用户ID |



\*\*响应数据\*\*:



```json

{

&#x20; "code": 200,

&#x20; "message": "操作成功",

&#x20; "data": 456,

&#x20; "timestamp": 1234567890

}

```



| 字段名  | 类型   | 说明   |

| ---- | ---- | ---- |

| data | Long | 会话ID |



\*\*错误响应\*\*:



```json

{

&#x20; "code": 500,

&#x20; "message": "不能与自己创建会话",

&#x20; "data": null,

&#x20; "timestamp": 1234567890

}

```



\*\*使用场景\*\*:



\- 用户点击某个用户的头像或昵称，准备发起聊天时

\- 需要先获取会话ID，然后使用该会话ID发送消息



\*\*\*



\### 2. 发送消息



\*\*接口地址\*\*: `POST /api/message/send`



\*\*功能说明\*\*: 发送私信消息



\*\*请求参数\*\*:



```json

{

&#x20; "chatId": 456,

&#x20; "receiverId": 123,

&#x20; "content": "你好，请问在吗？"

}

```



| 参数名        | 类型     | 必填 | 说明                  |

| ---------- | ------ | -- | ------------------- |

| chatId     | Long   | 是  | 会话ID（通过获取或创建会话接口获得） |

| receiverId | Long   | 是  | 接收者用户ID             |

| content    | String | 是  | 消息内容，不能为空           |



\*\*响应数据\*\*:



```json

{

&#x20; "code": 200,

&#x20; "message": "操作成功",

&#x20; "data": 789,

&#x20; "timestamp": 1234567890

}

```



| 字段名  | 类型   | 说明     |

| ---- | ---- | ------ |

| data | Long | 新消息的ID |



\*\*错误响应\*\*:



```json

{

&#x20; "code": 500,

&#x20; "message": "会话不存在",

&#x20; "data": null,

&#x20; "timestamp": 1234567890

}

```



\*\*使用场景\*\*:



\- 用户在聊天界面输入消息后点击发送

\- 建议先调用"获取或创建会话"接口获取chatId，再调用此接口发送消息



\*\*\*



\### 3. 获取聊天会话列表



\*\*接口地址\*\*: `POST /api/message/chat/list`



\*\*功能说明\*\*: 获取当前用户的所有聊天会话列表，按更新时间倒序排列



\*\*请求参数\*\*:



```json

{

&#x20; "pageNum": 1,

&#x20; "pageSize": 10

}

```



| 参数名      | 类型      | 必填 | 默认值 | 说明   |

| -------- | ------- | -- | --- | ---- |

| pageNum  | Integer | 否  | 1   | 页码   |

| pageSize | Integer | 否  | 10  | 每页数量 |



\*\*响应数据\*\*:



```json

{

&#x20; "code": 200,

&#x20; "message": "操作成功",

&#x20; "data": {

&#x20;   "list": \[

&#x20;     {

&#x20;       "chatId": 456,

&#x20;       "otherUserId": 123,

&#x20;       "otherNickname": "张三",

&#x20;       "otherAvatar": "https://example.com/avatar.jpg",

&#x20;       "lastMessageId": 789,

&#x20;       "lastMessageContent": "你好，请问在吗？",

&#x20;       "lastMessageTime": "2024-01-15T10:30:00",

&#x20;       "updateTime": "2024-01-15T10:30:00",

&#x20;       "unreadCount": 2

&#x20;     }

&#x20;   ],

&#x20;   "total": 10,

&#x20;   "pageNum": 1,

&#x20;   "pageSize": 10,

&#x20;   "totalPages": 1,

&#x20;   "hasNextPage": false,

&#x20;   "hasPreviousPage": false

&#x20; },

&#x20; "timestamp": 1234567890

}

```



\*\*ChatVO 数据结构\*\*:



| 字段名                | 类型            | 说明       |

| ------------------ | ------------- | -------- |

| chatId             | Long          | 会话ID     |

| otherUserId        | Long          | 对方用户ID   |

| otherNickname      | String        | 对方昵称     |

| otherAvatar        | String        | 对方头像URL  |

| lastMessageId      | Long          | 最后一条消息ID |

| lastMessageContent | String        | 最后一条消息内容 |

| lastMessageTime    | LocalDateTime | 最后一条消息时间 |

| updateTime         | LocalDateTime | 会话更新时间   |

| unreadCount        | Integer       | 未读消息数量   |



\*\*PageVO 数据结构\*\*:



| 字段名             | 类型      | 说明     |

| --------------- | ------- | ------ |

| list            | List    | 数据列表   |

| total           | Long    | 总记录数   |

| pageNum         | Integer | 当前页码   |

| pageSize        | Integer | 每页数量   |

| totalPages      | Integer | 总页数    |

| hasNextPage     | Boolean | 是否有下一页 |

| hasPreviousPage | Boolean | 是否有上一页 |



\*\*使用场景\*\*:



\- 用户进入私信页面，显示所有聊天会话列表

\- 下拉刷新或上拉加载更多会话



\*\*\*



\### 4. 获取消息列表



\*\*接口地址\*\*: `POST /api/message/list`



\*\*功能说明\*\*: 获取指定会话的消息列表，按创建时间正序排列



\*\*请求参数\*\*:



```json

{

&#x20; "chatId": 456,

&#x20; "pageNum": 1,

&#x20; "pageSize": 20

}

```



| 参数名      | 类型      | 必填 | 默认值 | 说明   |

| -------- | ------- | -- | --- | ---- |

| chatId   | Long    | 是  | -   | 会话ID |

| pageNum  | Integer | 否  | 1   | 页码   |

| pageSize | Integer | 否  | 20  | 每页数量 |



\*\*响应数据\*\*:



```json

{

&#x20; "code": 200,

&#x20; "message": "操作成功",

&#x20; "data": {

&#x20;   "list": \[

&#x20;     {

&#x20;       "id": 789,

&#x20;       "chatId": 456,

&#x20;       "senderId": 1,

&#x20;       "senderNickname": "李四",

&#x20;       "senderAvatar": "https://example.com/avatar1.jpg",

&#x20;       "receiverId": 123,

&#x20;       "receiverNickname": "张三",

&#x20;       "receiverAvatar": "https://example.com/avatar2.jpg",

&#x20;       "content": "你好，请问在吗？",

&#x20;       "isRead": 1,

&#x20;       "createTime": "2024-01-15T10:30:00"

&#x20;     }

&#x20;   ],

&#x20;   "total": 50,

&#x20;   "pageNum": 1,

&#x20;   "pageSize": 20,

&#x20;   "totalPages": 3,

&#x20;   "hasNextPage": true,

&#x20;   "hasPreviousPage": false

&#x20; },

&#x20; "timestamp": 1234567890

}

```



\*\*MessageVO 数据结构\*\*:



| 字段名              | 类型            | 说明               |

| ---------------- | ------------- | ---------------- |

| id               | Long          | 消息ID             |

| chatId           | Long          | 会话ID             |

| senderId         | Long          | 发送者用户ID          |

| senderNickname   | String        | 发送者昵称            |

| senderAvatar     | String        | 发送者头像URL         |

| receiverId       | Long          | 接收者用户ID          |

| receiverNickname | String        | 接收者昵称            |

| receiverAvatar   | String        | 接收者头像URL         |

| content          | String        | 消息内容             |

| isRead           | Integer       | 是否已读（0:未读, 1:已读） |

| createTime       | LocalDateTime | 消息创建时间           |



\*\*使用场景\*\*:



\- 用户点击某个会话，进入聊天详情页面

\- 上拉加载更多历史消息

\- 注意：调用此接口后，该会话中当前用户的未读消息会被自动标记为已读



\*\*\*



\### 5. 获取未读消息数量



\*\*接口地址\*\*: `POST /api/message/unread/count`



\*\*功能说明\*\*: 获取当前用户的未读消息总数



\*\*请求参数\*\*: 无（请求体传空对象 `{}` 即可）



\*\*响应数据\*\*:



```json

{

&#x20; "code": 200,

&#x20; "message": "操作成功",

&#x20; "data": 5,

&#x20; "timestamp": 1234567890

}

```



| 字段名  | 类型   | 说明     |

| ---- | ---- | ------ |

| data | Long | 未读消息总数 |



\*\*使用场景\*\*:



\- 在导航栏或消息图标上显示未读消息数量

\- 定时轮询更新未读消息数量（建议使用WebSocket实时推送）

\## 完整使用流程示例



\### 场景1: 发起新聊天



```javascript

// 1. 先获取或创建会话

const chatResponse = await fetch('/api/message/chat/get-or-create', {

&#x20; method: 'POST',

&#x20; headers: {

&#x20;   'Content-Type': 'application/json',

&#x20;   'Authorization': 'Bearer YOUR\_TOKEN'

&#x20; },

&#x20; body: JSON.stringify({

&#x20;   otherUserId: 123

&#x20; })

});

const chatResult = await chatResponse.json();

const chatId = chatResult.data;



// 2. 使用会话ID发送消息

const sendResponse = await fetch('/api/message/send', {

&#x20; method: 'POST',

&#x20; headers: {

&#x20;   'Content-Type': 'application/json',

&#x20;   'Authorization': 'Bearer YOUR\_TOKEN'

&#x20; },

&#x20; body: JSON.stringify({

&#x20;   chatId: chatId,

&#x20;   receiverId: 123,

&#x20;   content: '你好，请问在吗？'

&#x20; })

});

const sendResult = await sendResponse.json();

console.log('消息发送成功，消息ID:', sendResult.data);

```



\### 场景2: 查看聊天列表

\## 注意事项



1\. \*\*会话ID的使用\*\*: 发送消息时建议传入`chatId`，这样可以提高查询效率

2\. \*\*消息已读状态\*\*: 调用"获取消息列表"接口后，该会话中当前用户的未读消息会自动标记为已读

3\. \*\*权限控制\*\*: 用户只能查看自己参与的会话和消息

4\. \*\*时间格式\*\*: 所有时间字段使用ISO 8601格式，如 `2024-01-15T10:30:00`

5\. \*\*分页参数\*\*: 分页从1开始，不是从0开始

6\. \*\*WebSocket推送\*\*: 建议配合WebSocket实现实时消息推送，减少轮询频率

\## 错误码说明



| 错误码 | 说明               |

| --- | ---------------- |

| 200 | 操作成功             |

| 500 | 服务器内部错误          |

| 其他  | 具体错误信息见message字段 |



常见错误信息：



\- "不能与自己创建会话"

\- "用户不存在"

\- "会话不存在"

\- "无权访问此会话"

\- "不能发送给自己"

\- "接收者不存在"

\- "消息内容不能为空"

\- "会话id不能为空"

\- "接收者ID不能为空"

\- "对方用户ID不能为空"
